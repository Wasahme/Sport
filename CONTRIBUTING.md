## 🚀 البدء في المساهمة

### 1. إعداد البيئة المحلية

#### المتطلبات المسبقة:
- Java 11 أو أحدث
- Android Studio أو Android SDK
- Git

#### خطوات الإعداد:
```bash
# 1. Fork المشروع على GitHub
# 2. Clone المشروع
git clone https://github.com/YOUR_USERNAME/smart-medicine-doctor.git
cd smart-medicine-doctor

# 3. إضافة remote للمشروع الأصلي
git remote add upstream https://github.com/ORIGINAL_OWNER/smart-medicine-doctor.git

# 4. تشغيل script الإعداد
./setup.sh  # على Linux/macOS
# أو
setup.bat   # على Windows
```

### 2. إنشاء فرع جديد
```bash
# تحديث الفرع الرئيسي
git checkout main
git pull upstream main

# إنشاء فرع جديد للميزة
git checkout -b feature/your-feature-name
# أو
git checkout -b fix/your-bug-fix
```

### 5. إرسال Pull Request

#### قبل إرسال PR:
- [ ] تأكد من أن الكود يعمل بشكل صحيح
- [ ] أضف اختبارات للميزات الجديدة
- [ ] حدث الوثائق إذا لزم الأمر
- [ ] تأكد من أن جميع الاختبارات تمر

#### إنشاء PR:
1. اذهب إلى صفحة المشروع على GitHub
2. اضغط "New Pull Request"
3. اختر الفرع الخاص بك
4. املأ نموذج PR:
   - **العنوان**: وصف مختصر للتغيير
   - **الوصف**: شرح مفصل للتغييرات
   - **النوع**: Bug fix / Feature / Documentation
   - **اختبار**: كيف تم اختبار التغييرات

## 📋 معايير المساهمة

### معايير الكود:
- **التسمية**: استخدم أسماء واضحة ومفهومة
- **التعليقات**: أضف تعليقات باللغة العربية للوظائف المعقدة
- **التنسيق**: استخدم تنسيق Kotlin القياسي
- **الأداء**: تأكد من أن الكود فعال

### معايير الرسائل:
```bash
# أمثلة على رسائل commit جيدة
feat: إضافة ميزة التذكير الصوتي
fix: إصلاح مشكلة في استخراج النصوص
docs: تحديث دليل التثبيت
style: تحسين تنسيق الكود
refactor: إعادة هيكلة كود معالجة الصور
test: إضافة اختبارات لوظيفة استخراج الأدوية
```

### معايير الوثائق:
- اكتب باللغة العربية
- استخدم أمثلة واضحة
- أضف صور توضيحية إذا لزم الأمر
- حدث جميع الملفات ذات الصلة

## 🐛 الإبلاغ عن الأخطاء

### عند الإبلاغ عن خطأ:
1. **ابحث أولاً** - تحقق من وجود تقارير مشابهة
2. **استخدم القالب** - املأ نموذج الإبلاغ عن الأخطاء
3. **أرفق معلومات مفيدة**:
   - إصدار التطبيق
   - نظام التشغيل
   - خطوات إعادة الإنتاج
   - Screenshots أو تسجيلات فيديو
   - رسائل الخطأ

### قالب الإبلاغ عن الأخطاء:
```markdown
## وصف المشكلة
وصف واضح ومختصر للمشكلة.

## خطوات إعادة الإنتاج
1. اذهب إلى '...'
2. اضغط على '...'
3. انتقل إلى '...'
4. شاهد الخطأ

## السلوك المتوقع
وصف لما كان يجب أن يحدث.

## Screenshots
أرفق صور توضيحية إذا لزم الأمر.

## معلومات النظام
- إصدار التطبيق: [مثال: 1.0.0]
- نظام التشغيل: [مثال: Android 12]
- الجهاز: [مثال: Samsung Galaxy S21]

## معلومات إضافية
أي معلومات أخرى ذات صلة.
```
