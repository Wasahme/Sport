# المساهمة في FitTracker

شكراً لك على اهتمامك بالمساهمة في تطبيق FitTracker! 🎉

## كيفية المساهمة

### الإبلاغ عن الأخطاء

إذا وجدت خطأ في التطبيق، يرجى:

1. البحث في [Issues](https://github.com/yourusername/fittracker/issues) للتأكد من عدم الإبلاغ عنه مسبقاً
2. إنشاء issue جديد مع:
   - وصف مفصل للمشكلة
   - خطوات إعادة إنتاج المشكلة
   - معلومات الجهاز ونظام التشغيل
   - لقطة شاشة إذا كان ذلك ممكناً

### اقتراح ميزات جديدة

لاقتراح ميزة جديدة:

1. ابحث في [Issues](https://github.com/yourusername/fittracker/issues) للتأكد من عدم اقتراحها مسبقاً
2. أنشئ issue جديد مع:
   - وصف مفصل للميزة
   - سبب الحاجة إليها
   - أمثلة على الاستخدام
   - أي أفكار إضافية

### المساهمة بالكود

#### إعداد بيئة التطوير

1. Fork المشروع
2. Clone المشروع محلياً:
   ```bash
   git clone https://github.com/YOUR_USERNAME/fittracker.git
   cd fittracker
   ```
3. افتح المشروع في Android Studio
4. تأكد من تثبيت جميع التبعيات

#### إرشادات التطوير

##### معايير الكود

- استخدم **Kotlin** كلغة البرمجة الأساسية
- اتبع **Clean Architecture** مع **MVVM**
- استخدم **Jetpack Compose** للواجهات
- اتبع **Material Design 3** للتصميم
- اكتب تعليقات باللغة العربية
- استخدم أسماء متغيرات ووظائف واضحة

##### هيكل المشروع

```
app/src/main/java/com/fittracker/app/
├── data/           # طبقة البيانات
│   ├── local/      # قاعدة البيانات المحلية
│   ├── remote/     # APIs والخدمات السحابية
│   └── repository/ # تنفيذ المستودعات
├── domain/         # طبقة المنطق التجاري
│   ├── model/      # نماذج البيانات
│   ├── repository/ # واجهات المستودعات
│   └── usecase/    # حالات الاستخدام
├── presentation/   # طبقة العرض
│   ├── ui/         # واجهات المستخدم
│   └── viewmodel/  # نماذج العرض
├── di/             # حقن التبعيات
└── utils/          # أدوات مساعدة
```

##### إرشادات Git

- استخدم رسائل commit واضحة ومختصرة
- اكتب رسائل commit باللغة العربية
- استخدم prefixes مناسبة:
  - `feat:` للميزات الجديدة
  - `fix:` لإصلاح الأخطاء
  - `docs:` للتوثيق
  - `style:` لتنسيق الكود
  - `refactor:` لإعادة هيكلة الكود
  - `test:` للاختبارات
  - `chore:` للمهام الروتينية

مثال:
```bash
git commit -m "feat: إضافة متتبع الماء مع تذكيرات"
git commit -m "fix: إصلاح مشكلة في عداد الخطوات"
```

#### عملية المساهمة

1. أنشئ فرع جديد للميزة:
   ```bash
   git checkout -b feature/اسم-الميزة
   ```

2. اكتب الكود مع اتباع المعايير

3. اكتب اختبارات للميزة الجديدة

4. تأكد من أن جميع الاختبارات تمر:
   ```bash
   ./gradlew test
   ```

5. تأكد من أن الكود يتبع معايير التنسيق:
   ```bash
   ./gradlew ktlintCheck
   ```

6. اكتب أو حدث التوثيق

7. Commit التغييرات:
   ```bash
   git add .
   git commit -m "feat: وصف الميزة المضافة"
   ```

8. Push الفرع:
   ```bash
   git push origin feature/اسم-الميزة
   ```

9. أنشئ Pull Request

#### إرشادات Pull Request

- اكتب عنوان واضح ومختصر
- اكتب وصف مفصل للتغييرات
- اربط PR بـ Issue إذا كان ذلك مناسباً
- أضف لقطات شاشة إذا كانت التغييرات تتعلق بالواجهة
- تأكد من أن جميع الاختبارات تمر
- اطلب مراجعة من المطورين

### المساهمة بالترجمة

للمساهمة في ترجمة التطبيق:

1. اذهب إلى ملف `app/src/main/res/values/strings.xml`
2. أضف الترجمات المطلوبة
3. تأكد من صحة الترجمة ووضوحها
4. أنشئ Pull Request

### المساهمة بالتصميم

للمساهمة في تحسين التصميم:

1. اتبع Material Design 3
2. تأكد من دعم الوضع الليلي والفاتح
3. تأكد من دعم اللغة العربية والـ RTL
4. أضف لقطات شاشة للتصميم المقترح
5. أنشئ Issue أو Pull Request
